# Android App UI 設計規劃

## 一、整體架構

```
App 入口（Splash Screen）
    │
    ├── 未登入 ──→ 登入畫面 ──→ 忘記密碼 / 首次設定密碼
    │
    └── 已登入 ──→ 主頁（Home Dashboard）
                    │
          ┌─────────┼──────────────────┐
          │         │                  │
      新增檢測    歷史紀錄         個人設定
      （主流程）
```

## 二、主流程畫面詳細設計（7個步驟）

### 步驟 1：登入畫面（LoginActivity）

**畫面元件：**
- Logo + 品牌標語
- Email 輸入框
- 密碼輸入框（含顯示/隱藏按鈕）
- 【登入】按鈕
- 「忘記密碼？」連結
- 「首次使用？聯絡我們取得帳號」說明文字

**資料驗證：**
- Email 格式驗證
- 密碼至少8字元
- 失敗3次後顯示「帳號已被暫時鎖定」

**技術重點：**
- JWT Token 儲存於 Android Keystore（加密安全儲存）
- 支援生物辨識登入（指紋/臉部辨識）- 下次自動登入

**技術整合（統一金流 ECPay）：**
```
付款流程：
1. App → 後端 API：建立訂單（POST /api/payments/create）
2. 後端 → ECPay：建立交易，取得付款網址
3. App → WebView：載入 ECPay 付款頁面
4. 使用者完成付款
5. ECPay → 後端（Webhook）：付款通知
6. 後端：更新點數 + 通知 App（WebSocket 或 FCM）
7. App：顯示「付款成功，點數已更新」→ 繼續流程
```

## 三、其他主要畫面

### 歷史紀錄（HistoryFragment）
- 按時間排列的所有檢測場次
- 篩選：受測者 / 報告類型 / 日期範圍
- 每筆點選 → 查看摘要 + 下載 PDF
- 長按 → 刪除選項

### 個人設定（SettingsFragment）
- 帳號資訊與密碼修改
- 點數餘額與購買紀錄
- 設備管理（已配對的腦波頭帶）
- 通知設定（報告生成通知）
- 隱私政策 / 使用條款
- 登出

## 四、技術架構建議

### Android 技術棧

| 類別 | 建議技術 |
|------|---------|
| 開發語言 | Kotlin |
| UI 框架 | Jetpack Compose（或 XML + ViewBinding）|
| 架構模式 | MVVM + Clean Architecture |
| 網路請求 | Retrofit 2 + OkHttp（JWT Auth Header）|
| 本地資料庫 | Room Database（暫存離線資料）|
| 藍牙連接 | Android BLE（Bluetooth Low Energy）API |
| 狀態管理 | StateFlow + ViewModel |
| 相依注入 | Hilt |
| PDF 顯示 | Android PdfRenderer 或 PdfViewer 套件 |
| 即時更新 | WebSocket（OkHttp WebSocket）或 Firebase FCM |
| 付款整合 | ECPay SDK（WebView 方式）|
| 圖表 | MPAndroidChart（腦波折線圖）|

### API 通訊格式

```json
// 開始檢測 Request
POST /api/sessions/start
{
  "subject_id": 123,
  "report_types": ["cognitive", "stress", "mbti"],
  "device_id": 456,
  "questionnaire_data": { ... }
}

// 上傳每分鐘腦波資料 Request
POST /api/sessions/{session_id}/captures
{
  "minute_index": 45,
  "delta_power": 12.3,
  "theta_power": 8.7,
  "alpha_power": 15.2,
  "beta_power": 11.4,
  "gamma_power": 6.8,
  "smr_power": 9.1,
  "fp1_alpha": 14.8,
  "fp2_alpha": 15.9,
  "attention_score": 72,
  "signal_quality": 0
}

// 結束檢測
POST /api/sessions/{session_id}/complete
{
  "total_captured": 180
}
```

### 離線容錯機制
- 腦波資料先存入 Room Database（本機）
- 每分鐘嘗試上傳至後端（有網路時）
- 網路恢復後自動同步離線資料
- 設備斷線 → 顯示警告，保留已採集資料
